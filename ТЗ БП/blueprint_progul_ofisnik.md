## Этап 3: Фиксация факта отсутствия

### Шаг 3.1 — Валидация и консультация по доказательствам

| Lane | Содержание |
|------|------------|
| **Руководитель** | Описывает как узнал об отсутствии. Предоставляет доказательства по инструкции |
| **Ассистент (фронт)** | Запрашивает подтверждение факта отсутствия. Выдаёт инструкцию: что принимается как доказательство и требования к оформлению |
| **Ассистент (система)** | Выгружает инструкции из DS-05. Проверяет: отсутствие > 4 ч подряд или весь рабочий день (иначе — не прогул) |
| **Источники данных** | DS-05: Инструкции по проверке доказательств. DS-01: ТК РФ ст. 81 п. 6а |
| **Эксперт кад. учёта** | — |

**Требования к доказательствам (из DS-05):**
- Переписка в мессенджере: полная (не выдержки), дата/время видны, идентификатор собеседника, + скриншот контактов из Стаффа
- Переписка по email: адреса ящиков видны, полный экран с датой/временем
- Выгрузка из системы: наименование системы + URL в адресной строке браузера
- Любой скриншот: качественный и читаемый

**Не засчитывается в прогул:** обеденное время, больничный, согласованный отпуск, выходные, праздники, командировка.

### Шаг 3.2 — Составление акта об отсутствии

| Lane | Содержание |
|------|------------|
| **Руководитель** | Предоставляет: дату и время отсутствия, ФИО + должность сотрудника (полное), ФИО + должности 3 свидетелей |
| **Ассистент (фронт)** | Запрашивает реквизиты для акта. Формирует акт по шаблону. Выдаёт готовый текст |
| **Ассистент (система)** | Подставляет данные в DS-03. Валидирует: свидетели из того же ЮЛ и того же офиса, в дату нарушения были на рабочем месте (не в отпуске/командировке/на больничном) |
| **Источники данных** | DS-03: Шаблон акта об отсутствии. DS-01: ТК РФ ст. 81 п. 6а |
| **Эксперт кад. учёта** | — |

**Ограничения по авторам акта (критично):**
- Минимум 3 свидетеля
- Все из того же ЮЛ и того же офиса
- В дату нарушения должны быть на рабочем месте

## Этап 4: Составление служебной записки

### Шаг 4.1 — Сбор данных и формирование СЗ

| Lane | Содержание |
|------|------------|
| **Руководитель** | Предоставляет: предшествующее поведение сотрудника, последствия для компании/подразделения, каналы связи с сотрудником и результат |
| **Ассистент (фронт)** | Запрашивает недостающие данные по структуре СЗ. Формирует СЗ по шаблону. Выдаёт готовый текст |
| **Ассистент (система)** | Подставляет данные в DS-04. Включает нарушенные НПА из DS-01. Ничего не додумывает — только факты от руководителя |
| **Источники данных** | DS-04: Шаблон СЗ. DS-01: ТК РФ (нарушенные статьи) |
| **Эксперт кад. учёта** | — |

**Структура СЗ (обязательные разделы):**
1. Шапка: кто, кому, организация, должность
2. Факт нарушения: что, когда точно (дата и время), где, кто (полное наименование + ФИО)
3. Обстоятельства нарушения
4. Предшествующее поведение сотрудника
5. Использованные каналы связи для выяснения причин
6. Нарушенные НПА (законодательство + ЛНА + пункты ТД)
7. Негативные последствия
8. Просьба провести проверку
9. Приложения: акты, скриншоты переписки, цитаты НПА

**Ограничения:**
- Автор СЗ — непосредственный руководитель из того же ЮЛ (редко — вышестоящий или функциональный)
