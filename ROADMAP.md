# Roadmap: HR-ассистент — план доработок

## Статус MVP
Реализован сценарий "Прогул офисник": диалог → проверка сроков → акт + СЗ (готовые .docx).
Интерфейс — Telegram-бот. Диалог на Claude Opus 4.8, генерация на Ollama qwen2.5:7b,
память и кейсы в Postgres, шаблоны в Google Drive.

---

## Открытые задачи

### ✅ Сделано

#### P0.1 — Telegram-бот + промежуточные ответы — ГОТОВО
- Встроенный n8n-чат заменён на Telegram (Telegram Trigger + Send Message).
- Промежуточные статусы ⏳ во время генерации («Анализирую нарушение…», «Готовлю документы…»).
- Приём фото-доказательств из Telegram (с валидацией) → запись в `hr_evidence`.
- Память диалога перенесена на **Postgres Chat Memory** (sessionKey = chat_id).
- Webhook через `cloudflared` туннель (`WEBHOOK_URL`).

#### P1.1 — Гибкая подгрузка шаблонов + рендер .docx — ГОТОВО
- Шаблоны вынесены в **Google Drive** (.docx с `{{плейсхолдерами}}`) — правятся без касания workflow.
- Рендер готового **.docx** через `adm-zip` (скачать шаблон → подставить значения → отправить файлом).
- Гибрид заполнения: детерминированные поля (Code) + «умные» поля от Ollama (JSON: падежи, абзацы).
- Вариативность документов по сценарию через `DOC_MAP` + развилки `Нужен акт?` / `Нужна СЗ?`.
- Чеклист тоже подгружается динамически из Drive (Google Doc → text/plain).

#### P1.4 (частично) — запись данных в Postgres — ГОТОВО (база)
- Таблицы `hr_cases` (кейсы) и `hr_evidence` (доказательства), схема в `db/schema.sql`.
- Запись кейса после успешной генерации. Аналитика/дашборды — позже.

#### Доводка сбора полей под шаблоны (прогул) — ГОТОВО
- Агент собирает **все 25 полей** под шаблоны: + город/адрес рабочего места, подразделение
  руководителя, **2 свидетеля** (с пояснением зачем). Передача `workplace`, `witnesses[]`,
  `manager.department` через `generate_documents`.
- Полный маппинг 25 плейсхолдеров в «Значения документов»; хардкод организации
  (`АО «ПФ «СКБ Контур»`), адресата СЗ (`Директору` + пустой ФИО), авто-время акта.
- Фикс потери данных: все far-ссылки после Telegram/HTTP-нод переведены на `.first()` (было `.item`,
  рвалось → развилки в False, `chat_id is empty`). См. memory `n8n-first-vs-item-far-refs`.

#### Генерация без LLM (детерминированная) — ГОТОВО
- Убран Ollama из активного потока генерации. Склонение ФИО (дательный падеж) — через `petrovich`
  + JS-fallback; обстоятельства и приложения — склейка кодом. Быстро, предсказуемо, без GPU.
- Островок нарушений (первый Ollama-вызов) выключен (`disabled`) как задел под RAG — он дублировал
  «обстоятельства» с кривой ссылкой на ТК и шёл в мусорку.
- Диалог переведён на Claude (нода `Ollama Chat Model` удалена).

#### Скриншоты-доказательства в СЗ — ГОТОВО
- Фото из Telegram скачивается → `hr_evidence.file_b64` (base64). Агент собирает `evidenceFiles[]`.
- При генерации СЗ фото вставляются в .docx через OOXML («Скриншот N»), якорь `{{скриншоты}}`.
- Уроки: расширение media = зарегистрированному в `[Content_Types].xml` (иначе Word «повреждён»);
  `chat_id = $1::text` в SQL (колонка TEXT, из n8n приходит число).

#### P2.1 — Деплой на сервер — ГОТОВО (пилот)
- Docker Compose (n8n + Postgres + Caddy) на **Timeweb VPS в Нидерландах** (Anthropic блочит РФ-IP).
- Постоянный домен **hr-kontur.ru** + авто-HTTPS (Caddy/Let's Encrypt) вместо cloudflared-туннеля.
- Образ n8n с `petrovich`; swap 2 ГБ (1 ГБ RAM); зеркало Docker Hub против rate-limit.
- Артефакты и one-shot `bootstrap.sh` — в `deploy/`. Уроки: SSH через VPN виснет (MTU black hole) →
  деплой через веб-консоль Timeweb / снижение MTU.

### 🟡 P1 — Важно (ограничивает функциональность)

#### P0.2 — LLM-robustness в нетиповых сценариях
**Проблема:** qwen2.5:7b путается при отклонении от happy path (неполные данные, граничные случаи).
**Что нужно:** проверить промпты на robustness, fallback-инструкции («нет данных → [УТОЧНИТЬ]»),
при необходимости — другая локальная модель. На диалоге сейчас Claude (отладка), снимает риск.

#### P1.2 — Реальный RAG по ЛНА и ИД
**Проблема:** Чеклист подгружается из Drive целиком, без поиска. При росте числа НПА — неуправляемо.
**Что нужно:**
- Векторная БД: Qdrant или pgvector в Postgres.
- Embedding: Ollama nomic-embed-text (уже установлен).
- PDF/docx → chunking → embeddings → топ-N чанков в промпт.
- Интеграция с RAG-AAS ЦИИ (Вова Поздняков, июль 2026).

#### P1.3 — Vision (прочтение артефактов)
**Проблема:** Vision-заглушка принимает только текстовое описание. Фото уже принимается из Telegram,
но не анализируется.
**Что нужно:**
- Ollama vision-модель: qwen2-vl или llava (`ollama pull`).
- HTTP Request к `127.0.0.1:11434/api/generate` с base64 images[].
- Модель описывает что видит → в промпт.
