# Roadmap: HR-ассистент — план доработок

## Статус MVP
Реализован сценарий "Прогул офисник": диалог → проверка сроков → акт + СЗ (готовые .docx
со скриншотами-доказательствами). Интерфейс — Telegram-бот (целевой канал — Mattermost, см. план).
Диалог на Claude Opus 4.8 (облако), генерация **детерминированная** (petrovich + JS, без LLM),
память и кейсы в Postgres, шаблоны в Google Drive. Развёрнут пилотом (Docker, Нидерланды).

---

## 🚦 Статус отладки (02.07.2026)

Крупный заход: переезд с Telegram/Anthropic на **Mattermost + KonturGPT внутри контура Контура**,
идентификация через Staff, относительные даты, генерация через Mattermost. Итог по факту:

**✅ ОТЛАЖЕНО вживую:**
- **Mattermost** — вход (бот на `mmpy_bot` слушает WS → webhook `/mattermost-in`) и выход (посты). Переезд с Telegram.
- **Паспорт** — получение OAuth-токена (Basic base64, `/connect/token`).
- **Переезд** — диалог на **KonturGPT** (не Anthropic), n8n на `n8n-common.testkontur.ru` (внутри контура).

**⚠️ НАПИСАНО, НЕ ПРОТЕСТИРОВАНО (нужен живой прогон):**
- **Идентификация через Staff** — workflow готов (q-поиск, Bearer из Паспорта), в агенте не гонялся.
- **Относительные даты** в `calc_deadlines` — логика проверена node'ом, в живом агенте не гонялась.
- **Генерация документов через Mattermost** — upload `/api/v4/files` → post; multipart не проверен вживую.
- **Postgres на контуре** — 🔴 ЗАБЛОКИРОВАН правами. Нужен пакет от DBA:
  `GRANT USAGE, CREATE ON SCHEMA hr_disciplinary_assistant`; `ALTER ROLE ... SET search_path`;
  whitelist подсетей подов n8n в `pg_hba.conf` (IP плавают 10.216/10.218/10.220.x). До этого БД падает.

## Открытые задачи

### ✅ Сделано

#### P0.1 — Telegram-бот + промежуточные ответы — ГОТОВО
- Встроенный n8n-чат заменён на Telegram (Telegram Trigger + Send Message).
- Промежуточные статусы ⏳ во время генерации («Анализирую нарушение…», «Готовлю документы…»).
- Приём фото-доказательств из Telegram (с валидацией) → запись в `hr_evidence`.
- Память диалога перенесена на **Postgres Chat Memory** (sessionKey = chat_id).
- Webhook через `cloudflared` туннель (`WEBHOOK_URL`).

#### P1.1 — Гибкая подгрузка шаблонов + рендер .docx — ГОТОВО
- Шаблоны вынесены в **Google Drive** (.docx с `{{плейсхолдерами}}`) — правятся без касания workflow.
- Рендер готового **.docx** через `adm-zip` (скачать шаблон → подставить значения → отправить файлом).
- Гибрид заполнения: детерминированные поля (Code) + «умные» поля от Ollama (JSON: падежи, абзацы).
- Вариативность документов по сценарию через `DOC_MAP` + развилки `Нужен акт?` / `Нужна СЗ?`.
- Чеклист тоже подгружается динамически из Drive (Google Doc → text/plain).

#### P1.4 (частично) — запись данных в Postgres — ГОТОВО (база)
- Таблицы `hr_cases` (кейсы) и `hr_evidence` (доказательства), схема в `db/schema.sql`.
- Запись кейса после успешной генерации. Аналитика/дашборды — позже.

#### Доводка сбора полей под шаблоны (прогул) — ГОТОВО
- Агент собирает **все 25 полей** под шаблоны: + город/адрес рабочего места, подразделение
  руководителя, **2 свидетеля** (с пояснением зачем). Передача `workplace`, `witnesses[]`,
  `manager.department` через `generate_documents`.
- Полный маппинг 25 плейсхолдеров в «Значения документов»; хардкод организации
  (`АО «ПФ «СКБ Контур»`), адресата СЗ (`Директору` + пустой ФИО), авто-время акта.
- Фикс потери данных: все far-ссылки после Telegram/HTTP-нод переведены на `.first()` (было `.item`,
  рвалось → развилки в False, `chat_id is empty`). См. memory `n8n-first-vs-item-far-refs`.

#### Генерация без LLM (детерминированная) — ГОТОВО
- Убран Ollama из активного потока генерации. Склонение ФИО (дательный падеж) — через `petrovich`
  + JS-fallback; обстоятельства и приложения — склейка кодом. Быстро, предсказуемо, без GPU.
- Островок нарушений (первый Ollama-вызов) выключен (`disabled`) как задел под RAG — он дублировал
  «обстоятельства» с кривой ссылкой на ТК и шёл в мусорку.
- Диалог переведён на Claude (нода `Ollama Chat Model` удалена).

#### Скриншоты-доказательства в СЗ — ГОТОВО
- Фото из Telegram скачивается → `hr_evidence.file_b64` (base64). Агент собирает `evidenceFiles[]`.
- При генерации СЗ фото вставляются в .docx через OOXML («Скриншот N»), якорь `{{скриншоты}}`.
- Уроки: расширение media = зарегистрированному в `[Content_Types].xml` (иначе Word «повреждён»);
  `chat_id = $1::text` в SQL (колонка TEXT, из n8n приходит число).

#### P2.1 — Деплой на сервер — ГОТОВО (пилот)
- Docker Compose (n8n + Postgres + Caddy) на **Timeweb VPS в Нидерландах** (Anthropic блочит РФ-IP).
- Постоянный домен **hr-kontur.ru** + авто-HTTPS (Caddy/Let's Encrypt) вместо cloudflared-туннеля.
- Образ n8n с `petrovich`; swap 2 ГБ (1 ГБ RAM); зеркало Docker Hub против rate-limit.
- Артефакты и one-shot `bootstrap.sh` — в `deploy/`. Уроки: SSH через VPN виснет (MTU black hole) →
  деплой через веб-консоль Timeweb / снижение MTU.

**Решение Олега:** берём полную интеграцию (вопреки совету кадровички про «ручной ввод сейчас»).

- Предусловие: выяснить полную схему полей Стаффа.

### 🟠 M3 — Чеклист в начало процесса
- Если руководитель не знает процедуру — сначала ознакомить с чеклистом, провести по
  обязательным действиям (связь по каналам, проверка в кадрах: больничный/отпуск), ПОТОМ собирать кейс.
- Выдавать как чек-бокс («не на больничном, не в отпуске, звонил, писал»). Инфа в чеклисте уже есть.

### 🟠 M4 — Доработка сбора фактуры (вопросы агента)
- **Умная проверка дат:** «сегодня/вчера/послезавтра» → конкретные даты (прогул обычно в прошлом).
- **Время отсутствия:** полдня/опоздание, не только целый день; «с какого времени», «до сих пор или нет».
- **«отсутствовал» vs «отсутствует»** (период «с X по настоящее время»).
- **Город + полный адрес** — тянуть и подтверждать.
- **Канал связи** (телефон/мессенджер) — спрашивать (важно для скриншотов в СЗ).
- **Документ. подтверждение** (больничный/справка) из чеклиста; нет → неуважительная; доп.вопрос про медпомощь.
- **Время работы по ТД** — НОВЫЙ вопрос; «не знаю» → отдел кадров (заявка 911 / 9 кабинет — уточнить);
  риск «сам определяет режим» → акт не поставить.
- **Последствия отсутствия** — НОВЫЙ обязательный вопрос для СЗ; предложить варианты (упущенные
  клиенты, перераспределение нагрузки, невыполнение плана, влияние на дисциплину); «привело» И «приведёт».
- **Подробный рассказ + валидация:** просить детализировать кейс, но не додумывать за руководителя.
- **Свидетелей — ТРОЕ**, не двое.

### 🟠 M5 — Две раздельные проверки сроков ст.193
- **Для акта:** составляется в день обнаружения; задним числом нельзя (подделка). Вышел и отработал → акт не оформить.
- **Для СЗ:** сроки привлечения (по нарушению / по обнаружению).
- Прогул → акт + СЗ; **дистант → только СЗ** (акта нет).
- **Позитив:** убрать конкретные даты, писать «срок не истёк, можно привлекать».
- **Негатив:** расшифровать по-человечески («прошло больше 5,5 мес»), даты убрать/расшифровать.
- ⚠️ Сверить буфер: 6 мес/1 мес (встреча) vs 5 мес 15 дн / 20 дн (калькулятор) — у Касмыниной.

### 🟠 M6 — Правки шаблонов документов
**Акт:**
- Время вместо даты в конце: время **окончания нарушения** (целый день → конец рабочего дня;
  опоздание → время прихода).
- Подразделение — **всё дерево отдела** (кроме департамента, исключения пока игнорируем).
- **Город обязателен** (полный адрес вплоть до города).

**СЗ:**
- «отсутствовал» (1 день) vs «отсутствует по настоящее время» (период, акты кооперируются).
- Больше конкретики + раздел последствий.
- 🐞 Баг: СЗ тащит все скриншоты подряд — поправить.

### 🟢 M7 — Приоритизация сценариев
- ТОП-3 для запуска: **отсутствие на дистанте, разовый прогул, опоздание**.
- Одно «сложное» на пробу: **неисполнение должностных обязанностей (НДО)**.
- Адаптация/наставничество → испытательный срок (отдельно, позже).
