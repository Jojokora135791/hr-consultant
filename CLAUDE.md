# HR-консультант — CLAUDE.md

## Что это

ИИ-ассистент для руководителей Контура по оформлению дисциплинарных документов (акт об отсутствии, служебная записка). Реализуется на **n8n** + **Ollama** (локально). Статус: активная разработка, архитектура v3.

## Архитектура v3 — AI Agent + переиспользуемые lib-блоки

```
Chat Trigger
    ↓
AI Agent (qwen2.5:7b)
    ├── Tool: check_dates         → lib_check_dates.json (сроки ст.193 ТК)
    └── Tool: generate_documents  → sc_progul_ochny.json
                                        ↓
                                    lib_rag.json          (формулировка нарушений)
                                    lib_build_act.json    (акт об отсутствии)
                                    lib_build_sz.json     (служебная записка)
```

**Принцип переиспользования:** lib_rag, lib_build_sz одинаковы для всех сценариев.
Новый сценарий = новый sc_*.json + новый чеклист в knowledge/.

**Принцип "не захардкоживать":** статьи ТК, чеклисты и шаблоны — в файлах на диске.
Code ноды читают их через `fs.readFileSync`. Меняешь файл — меняется поведение.

## Настройка и запуск

### 1. Создать Credential: Ollama API
n8n → Settings → Credentials → Add → Ollama API
- Base URL: `http://127.0.0.1:11434`
> ⚠️ Именно `127.0.0.1`, не `localhost` — macOS резолвит в IPv6

### 2. Создать Credential: PostgreSQL
n8n → Settings → Credentials → Add → PostgreSQL
- Host: `localhost`, Port: `5432`, DB: `hr_assistant`, User: `olegkluev`

### 3. Импортировать workflow (порядок важен)
1. `lib/lib_check_dates.json` → заменить `REPLACE_ME` на ID PostgreSQL credential
2. `lib/lib_rag.json`
3. `lib/lib_build_act.json`
4. `lib/lib_build_sz.json`
5. `scenarios/sc_progul_ochny.json`
6. `00_main_agent.json` → заменить `REPLACE_ME_OLLAMA` на ID Ollama credential

### 4. Тест
Открой `HR-ассистент (AI Agent)` → Test Workflow → напиши: `"Помоги оформить прогул"`

## Как добавить новый сценарий

1. Создать `knowledge/{scenario}_checklist.md` — чеклист из .docx
2. Создать `templates/sz_{scenario}.md` — если нужен другой шаблон СЗ
3. Создать `scenarios/sc_{scenario}.json` — оркестратор (по образцу sc_progul_ochny)
4. Добавить в 00_main_agent.json новый tool или расширить generate_documents

## Ключевые ограничения ассистента

- Никогда ничего не додумывает — только факты от руководителя
- Граница: до передачи пакета документов эксперту включительно
- Если ситуация не под сценарий → направить к Касмыниной О. или Черепановой Т.
- Если нарушение связано с ИБ / коммерческой тайной → юр. служба / ОИБО

## Роли в процессе

| Роль | Описание |
|------|----------|
| Руководитель | Пользователь ассистента. Инициирует, предоставляет факты |
| ИИ-ассистент | Ведёт диалог, консультирует, формирует документы |
| Эксперт кадрового учёта | Черепанова Т., Касмынина О. Принимает финальный пакет |
| ОКУ | Получает табельные данные (НН), приказы |
